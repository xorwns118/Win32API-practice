Win32API 객체 및 기초 수학 스터디 가이드
===

퀴즈
---

1. 씬 매니저의 역할은 무엇이며, 씬 관리는 어떤 방식으로 이루어지나요?<br/>
2. 부모 클래스인 Object의 업데이트 함수가 가상 함수(virtual function)로 선언되어야 하는 이유는 무엇인가요?<br/>
3. 추상 클래스(abstract class)의 특징과 Object 클래스를 추상 클래스로 만들어야 하는 이유는 무엇인가요?<br/>
4. 자식 클래스에서 부모 클래스의 protected 멤버 변수에 직접 접근할 수 없는 이유는 무엇이며, 어떻게 접근해야 하나요?<br/>
5. 지역 변수와 멤버 변수의 차이점과 업데이트 함수 내에서 오브젝트의 위치를 변경할 때 주의해야 할 점은 무엇인가요?<br/>
6. 벡터의 정규화(normalization)는 무엇이며, 그 목적은 무엇인가요?<br/>
7. 각도(degree)와 라디안(radian)의 관계를 설명하고, 삼각함수에서 라디안 단위를 사용하는 이유는 무엇인가요?<br/>
8. 원주율(π)의 의미를 설명하고, 원의 둘레와 반지름 사이의 관계를 라디안을 이용하여 설명해 주세요.<br/>
9. 삼각함수(sin, cos, tan)의 기본적인 정의를 직각삼각형의 변의 길이를 이용하여 설명해 주세요.<br/>
10. 단위 원(unit circle)에서 각 Θ에 대한 좌표 (x, y)는 삼각함수와 어떤 관계를 가지나요?<br/>

퀴즈 정답
---
1. 씬 매니저는 게임 내 여러 씬들을 생성하고 관리하는 역할을 합니다. 현재 활성화된 씬을 지정하고, 해당 씬에 속한 오브젝트들의 업데이트와 렌더링 로직을 매 프레임마다 순회하며 호출하는 방식으로 씬 관리가 이루어집니다.<br/>
2. Object 클래스를 상속받는 다양한 자식 클래스(예: Player, Monster)는 각자 고유한 업데이트 로직을 가져야 합니다. 부모 클래스의 업데이트 함수를 가상 함수로 선언함으로써, 런타임 시 실제 객체의 타입에 맞는 오버라이딩된 업데이트 함수가 호출될 수 있도록 다형성을 구현하기 위함입니다.<br/>
3. 추상 클래스는 하나 이상의 순수 가상 함수를 포함하며, 객체를 직접 생성할 수 없는 클래스입니다. Object 클래스를 추상 클래스로 만들어, 모든 게임 오브젝트가 Object를 상속받아 구체적인 형태를 가지도록 강제하고, Object 자체로는 객체 생성을 막아 불필요한 기본 Object 객체 생성을 방지하기 위함입니다.<br/>
4. protected 접근 제한자는 해당 클래스와 자식 클래스에서는 접근이 가능하지만, 외부에서는 접근을 막습니다. 자식 클래스에서 부모의 protected 멤버 변수에 접근하기 위해서는 부모 클래스에서 해당 멤버 변수에 접근하거나 수정할 수 있는 public 또는 protected 접근 제한자를 가진 멤버 함수(getter/setter)를 제공해야 합니다.<br/>
5. 지역 변수는 함수 내에서 선언되고 해당 함수가 종료되면 소멸되는 반면, 멤버 변수는 클래스 내에 선언되어 객체가 존재하는 동안 유지됩니다. 업데이트 함수 내에서 지역 변수에 위치 정보를 변경하더라도 실제 오브젝트의 위치는 변경되지 않으므로, 변경된 위치 정보를 멤버 변수에 반영하기 위해 setter 함수 등을 사용하여 값을 갱신해야 합니다.<br/>
6. 벡터의 정규화는 벡터의 방향은 유지하면서 길이를 1로 만드는 과정입니다. 주로 방향 정보만을 필요로 할 때 사용되며, 벡터의 크기에 영향을 받지 않고 일관된 방향성을 표현하거나, 다른 벡터와의 연산을 단순화하는 데 목적이 있습니다.<br/>
7. 360도가 2π 라디안에 해당하며, π 라디안은 180도입니다. 삼각함수에서 라디안 단위를 사용하는 이유는 수학적 계산 및 미적분학적 성질이 더 깔끔하게 표현되고, 호의 길이와 반지름 사이의 직접적인 비율 관계를 가지기 때문에 다양한 공학 및 물리 분야에서 자연스럽게 사용되기 때문입니다.<br/>
8. 원주율(π)은 원의 둘레와 지름의 비율을 나타내는 상수입니다. 반지름이 r인 원의 둘레는 2πr로 표현될 수 있습니다. 라디안으로 한 바퀴는 2π 라디안이므로, 반지름에 라디안 각도를 곱하면 해당 각도에 해당하는 호의 길이를 얻을 수 있습니다.<br/>
9. 직각삼각형에서 특정 각 Θ에 대해, sinΘ는 빗변에 대한 대변의 비율, cosΘ는 빗변에 대한 인접변의 비율, tanΘ는 인접변에 대한 대변의 비율을 나타냅니다.<br/>
10. 반지름이 1인 단위 원에서 중심을 원점으로 하고 각 Θ만큼 회전한 점의 좌표 (x, y)는 x = cosΘ, y = sinΘ로 나타낼 수 있습니다. 이는 삼각함수가 원 위의 점의 좌표와 밀접하게 관련되어 있음을 보여줍니다.<br/>

용어 사전
---

1. ***씬***(Scene): 게임의 독립적인 화면 또는 단계를 의미합니다. 각 씬은 자체적인 오브젝트와 로직을 가집니다.<br/>
2. ***씬 매니저***(Scene Manager): 게임 내의 여러 씬들을 생성, 관리, 전환하는 역할을 담당하는 객체 또는 시스템입니다.<br/>
3. ***오브젝트***(Object): 게임을 구성하는 모든 요소(캐릭터, 배경, 아이템 등)의 기본 단위입니다.<br/>
4. ***업데이트***(Update): 매 프레임마다 오브젝트의 상태를 갱신하고 로직을 수행하는 함수 또는 과정입니다.<br/>
5. ***렌더링***(Rendering): 게임 화면에 오브젝트를 시각적으로 그리는 함수 또는 과정입니다.<br/>
6. ***가상 함수***(Virtual Function): 부모 클래스에서 선언되고 자식 클래스에서 오버라이딩될 수 있는 함수입니다. 런타임 시 실제 객체의 타입에 따라 호출될 함수가 결정됩니다 (다형성).<br/>
7. ***추상 클래스***(Abstract Class): 하나 이상의 순수 가상 함수를 포함하는 클래스로, 객체를 직접 생성할 수 없습니다. 상속을 통해 구체적인 자식 클래스를 만들어 사용해야 합니다.<br/>
8. ***순수 가상 함수***(Pure Virtual Function): 가상 함수이지만 부모 클래스에서 구현이 제공되지 않고, 반드시 자식 클래스에서 오버라이딩하여 구현해야 하는 함수입니다 (virtual 반환타입 함수명(매개변수) = 0;).<br/>
9. ***상속***(Inheritance): 한 클래스가 다른 클래스의 속성(멤버 변수, 멤버 함수)을 물려받아 사용할 수 있도록 하는 객체 지향 프로그래밍의 특징입니다.<br/>
10. ***다형성***(Polymorphism): 하나의 인터페이스나 부모 클래스 타입으로 여러 종류의 자식 클래스 객체를 참조하고, 런타임 시 실제 객체의 타입에 맞는 메서드를 호출할 수 있는 능력입니다.<br/>
11. ***접근 제한자***(Access Specifier): 클래스 멤버의 접근 범위를 제어하는 키워드입니다 (public, protected, private).<br/>
12. ***지역 변수***(Local Variable): 함수 내에서 선언되고 해당 함수 내에서만 유효한 변수입니다.<br/>
13. ***멤버 변수***(Member Variable): 클래스 내에 선언되어 객체의 속성을 나타내는 변수입니다.<br/>
14. ***싱글톤***(Singleton): 프로그램 내에서 하나의 인스턴스만 생성되도록 보장하는 디자인 패턴입니다.<br/>
15. ***벡터***(Vector): 크기와 방향을 동시에 가지는 양을 나타냅니다. 게임 개발에서 오브젝트의 이동 방향, 속도, 힘 등을 표현하는 데 사용됩니다.<br/>
16. ***스칼라***(Scalar): 크기만 가지고 방향은 가지지 않는 양을 나타냅니다. 속도의 크기, 질량, 온도 등이 스칼라의 예시입니다.<br/>
17. ***벡터의 정규화***(Vector Normalization): 벡터의 방향을 유지하면서 길이를 1로 만드는 과정입니다.<br/>
18. ***각도***(Degree): 원을 360등분하여 각의 크기를 나타내는 단위입니다.<br/>
19. ****라디안****(Radian): 원의 반지름과 호의 길이가 같아지는 각도를 1 라디안으로 정의하는 각도의 단위입니다.<br/>
20. ***원주율***(π): 원의 둘레와 지름의 비율을 나타내는 수학 상수 (약 3.141592).<br/>
21. ***삼각함수***(Trigonometric Functions): 각의 크기와 직각삼각형의 변의 길이 사이의 관계를 나타내는 함수 (sin, cos, tan 등).<br/>
22. ***단위 원***(Unit Circle): 반지름의 길이가 1인 원으로, 삼각함수의 값을 시각적으로 이해하는 데 유용합니다.<br/>
23. ***델타 타임***(Delta Time, dt): 이전 프레임이 완료되는 시점부터 현재 프레임이 시작되는 시점까지의 시간 간격을 의미합니다. 프레임 독립적인 움직임을 구현하는 데 사용됩니다.<br/>
