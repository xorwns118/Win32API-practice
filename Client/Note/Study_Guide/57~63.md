학습 가이드: Win32API를 활용한 게임 개발 기초
===

퀴즈
---

1. 알파 값이란 무엇이며, 0과 255일 때 각각 투명도는 어떻게 되나요?<br/>
2. BMP 포맷 이미지로 알파 블렌딩을 바로 할 수 없는 이유는 무엇인가요?<br/>
3. 알파 블렌딩 시 두 색상이 혼합되는 기본적인 공식은 무엇인가요?<br/>
4. Win32API에서 알파 블렌딩 기능을 수행하는 함수는 무엇인가요?<br/>
5. 카메라 페이드 아웃 효과를 구현할 때, 검은색 텍스처의 알파 값을 어떻게 변화시켜야 하나요?<br/>
6. 여러 카메라 이펙트를 순차적으로 실행하기 위해 어떤 데이터 구조를 사용할 수 있으며, 그 이유는 무엇인가요?<br/>
7. FSM에서 '상태(State)'와 '상태 전환(Transition)'은 각각 무엇을 의미하나요?<br/>
8. FSM 기반 AI에서 상태 전환을 즉시 처리하지 않고 이벤트 기반으로 처리하는 주된 이유는 무엇인가요?<br/>
9. 타일 맵 렌더링 시 모든 타일을 그리지 않고 일부만 그리는 렌더링 최적화 기법은 무엇인가요?<br/>
10. 타일 맵에서 화면에 보이는 영역의 타일만 렌더링하기 위해 어떤 정보를 활용해야 하나요?<br/>

퀴즈 정답
---

1. 알파 값은 이미지 픽셀의 투명도를 나타내는 값입니다. 0은 완전 투명하여 뒤의 배경이 그대로 보이며, 255는 완전 불투명하여 해당 픽셀의 색상만 보입니다.<br/>
2. 일반적인 BMP 포맷은 픽셀당 RGB 색상 정보만 저장하고 투명도를 위한 알파 값을 저장하는 알파 채널이 포함되어 있지 않기 때문입니다. 알파 블렌딩에는 알파 값이 필요합니다.<br/>
3. 최종 색상은 출력하려는 물체의 색상에 알파 비율을 곱한 값과 이미 그려진 배경 색상에 (1 - 알파 비율)을 곱한 값을 더하여 계산됩니다. 최종 색상 = (출력 색상 * 알파 비율) + (배경 색상 * (1 - 알파 비율))입니다.<br/>
4. Win32API에서 알파 블렌딩 기능을 수행하는 함수는 AlphaBlend 함수입니다.<br/>
5. 카메라 페이드 아웃 효과를 구현할 때는 검은색 텍스처를 화면에 덧그리며 점차 어둡게 만듭니다. 이를 위해 검은색 텍스처의 알파 값을 0(완전 투명)에서 255(불투명)로 점차 증가시켜야 합니다.<br/>
6. 여러 카메라 이펙트를 순차적으로 실행하기 위해 std::list와 같은 컨테이너를 사용하여 효과 정보를 저장할 수 있습니다. 리스트는 요소의 삽입/삭제가 효율적이어서 효과가 끝나면 리스트에서 쉽게 제거하고 다음 효과를 처리하기에 적합합니다.<br/>
7. FSM에서 '상태'는 몬스터가 특정 시점에 수행하는 행동 양식(예: 가만히 있기, 쫓아가기)을 의미하고, '상태 전환'은 어떤 조건이 만족되었을 때 하나의 상태에서 다른 상태로 바뀌는 과정을 의미합니다.<br/>
8. 상태 전환을 즉시 처리하면 한 프레임 내에서 여러 로직이 같은 몬스터의 상태를 다르게 인지할 수 있습니다. 이벤트 기반으로 처리하면 상태 변경 요청을 이벤트 큐에 등록하고 다음 프레임 시작 시점에 일괄 처리하여 모든 로직이 통일된 상태 정보를 가지고 동작하게 함으로써 동기화 문제를 해결하고 안정성을 높입니다.<br/>
9. 타일 맵 렌더링 시 모든 타일을 그리지 않고 화면에 보이는 영역의 타일만 그리는 렌더링 최적화 기법은 컬링(Culling) 또는 시야 절두체 컬링(Frustum Culling)과 유사한 개념을 적용한 것입니다.<br/>
10. 화면에 보이는 영역의 타일만 렌더링하기 위해서는 카메라의 현재 위치, 화면의 해상도, 그리고 타일 맵의 구조 정보(타일 크기, 가로/세로 타일 개수)를 활용해야 합니다. 이를 통해 화면 좌상단 및 우하단에 해당하는 타일의 인덱스 범위를 계산합니다.<br/>

에세이 형식 질문
---

1. Win32API의 AlphaBlend 함수를 사용하여 이미지의 투명도를 제어하는 방법을 상세히 설명하고, BLENDFUNCTION 구조체의 주요 멤버들이 투명도 표현에 어떻게 기여하는지 설명하시오.<br/>
2. 카메라 페이드 효과를 FSM 구조의 상태 전환 개념과 연관 지어 설명하고, 페이드 인/아웃 효과의 시간 진행에 따른 알파 값 변화 로직을 상세히 기술하시오.<br/>
3. 몬스터 AI를 FSM으로 설계할 때 필요한 주요 상태(State)들을 예시로 들고, 각 상태의 역할과 상태 간 전환이 발생하는 트리거(조건)를 구체적인 게임 상황과 연결하여 설명하시오.<br/>
4. 타일 맵 렌더링 최적화 기법의 필요성을 설명하고, 카메라 위치와 화면 해상도를 활용하여 화면 안에 들어오는 타일의 범위를 계산하는 로직 및 발생할 수 있는 경계 조건 예외 처리에 대해 상세히 기술하시오.<br/>
5. FSM 기반 AI 구현에서 상태 전환 요청을 이벤트 기반으로 처리하는 것의 장점과, 이러한 이벤트 처리 시스템이 전체 게임 루프(Game Loop)의 동기화 및 안정성에 미치는 영향에 대해 논하시오.<br/>

용어 설명 (Glossary)
---

***알파 블렌딩*** (Alpha Blending): 두 색상을 알파 값(투명도)을 기준으로 혼합하여 투명 또는 반투명 효과를 구현하는 기술.<br/>
***알파 채널*** (Alpha Channel): 이미지 데이터에서 투명도 정보를 저장하는 부분. RGB 색상 정보에 추가됨.<br/>
***픽셀 셰이더*** (Pixel Shader): 3D 그래픽스 파이프라인의 한 단계로, 픽셀 단위로 색상 및 기타 속성을 계산하는 프로그래밍 가능한 함수. Win32API GDI보다 더 정밀하고 유연한 픽셀 조작이 가능함.<br/>
***GPU*** (Graphics Processing Unit): 그래픽스 연산에 특화된 하드웨어 프로세서. 3D 렌더링, 셰이더 처리 등을 담당하여 CPU의 부하를 덜어줌.<br/>
***카메라 이펙트*** (Camera Effect): 게임 화면 전체 또는 일부에 적용되는 시각 효과 (예: 페이드, 화면 흔들림 등).<br/>
***페이드 인*** (Fade In): 화면이 점차 밝아지는 효과.<br/>
***페이드 아웃*** (Fade Out): 화면이 점차 어두워지는 효과.<br/>
***FSM*** (Finite State Machine): 유한한 개수의 상태와 상태 전환 규칙으로 시스템의 동작을 모델링하는 기법.<br/>
***상태*** (State): FSM에서 시스템이 특정 시점에 가지는 상태 (예: Idle, Trace, Attack).<br/>
***상태 전환*** (Transition): FSM에서 하나의 상태에서 다른 상태로 바뀌는 것. 특정 조건(트리거)에 의해 발생.<br/>
***트리거*** (Trigger): FSM에서 상태 전환을 유발하는 조건 또는 이벤트.<br/>
***팩토리 패턴*** (Factory Pattern): 객체 생성 로직을 캡슐화하여 객체 생성 과정을 추상화하는 디자인 패턴. 자주 사용되는 객체 조합이나 설정을 생성하는 함수를 모아놓는 데 사용될 수 있음.<br/>
***렌더링 최적화*** (Rendering Optimization): 게임 화면을 그리는 과정에서 불필요한 연산이나 함수 호출을 줄여 성능을 향상시키는 기법.<br/>
***뷰포트*** (Viewport): 카메라가 보고 있는 실제 게임 월드 영역에 해당하는 화면상의 영역.<br/>
***GDI*** (Graphics Device Interface): Windows 운영체제에서 그래픽 출력을 담당하는 API. Win32API의 그래픽 함수들이 GDI를 기반으로 동작함.<br/>
***타일 맵*** (Tile Map): 격자 형태로 배치된 작은 이미지(타일)들을 조합하여 게임 맵을 구성하는 방식.<br/>
