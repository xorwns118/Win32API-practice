Win32API 유니티 빌드 및 툴 개발 스터디 가이드
===

퀴즈
---

1. 잦은 빌드로 인해 빌드 속도가 느려지는 주된 이유는 무엇인가요?<br/>
2. 전방 선언의 사용이 컴파일 속도 향상에 도움이 되는 이유는 무엇인가요?<br/>
3. 유니티 빌드란 무엇이며, 빌드 속도를 어떻게 향상시키나요?<br/>
4. 비주얼 스튜디오에서 유니티 빌드 기능을 활성화하는 방법은 무엇인가요?<br/>
5. 툴 개발에서 '씬'이란 어떤 개념으로 사용되나요?<br/>
6. 타일 기반 게임에서 '타일'이란 무엇을 의미하며, 렌더링 방식은 기존 오브젝트와 어떻게 다른가요?<br/>
7. 툴 씬에서 타일을 효율적으로 관리하기 위해 타일 이미지를 모아 하나의 텍스처로 사용하는 이유는 무엇인가요?<br/>
8. 타일 텍스처 내에서 특정 타일 이미지를 선택하기 위해 인덱스를 사용하는 방식은 무엇인가요?<br/>
9. 모달 방식의 윈도우와 모달리스 방식의 윈도우의 주요 차이점은 무엇인가요?<br/>
10. 윈도우 프로시저(Window Procedure)란 무엇이며, 다이얼로그 박스에서 어떤 역할을 하나요?<br/>

퀴즈 정답
---

1. 파일 단위로 컴파일과 링크를 개별적으로 진행하고, 헤더 파일 간의 상호 참조로 인해 컴파일러가 불필요한 파일까지 연쇄적으로 확인해야 하기 때문입니다. 또한, 파일 개수가 많아질수록 컴파일러의 작업량이 증가합니다.<br/>
2. 헤더 파일에서 다른 헤더 파일을 직접 참조하는 대신 전방 선언을 사용하면 특정 파일에서 필요한 정보만을 포함하여 불필요한 헤더 포함으로 인한 컴파일 시간 증가를 막을 수 있습니다. 컴파일러가 연쇄적으로 확인해야 할 내용을 줄여줍니다.<br/>
3. 유니티 빌드는 여러 개의 소스 파일(.cpp)을 빌드 시에 하나의 임시 cpp 파일로 합쳐서 컴파일하는 방식입니다. 이렇게 하면 컴파일러가 한 번에 많은 코드를 처리하므로 파일 개별 컴파일 및 링크 과정에서 발생하는 오버헤드를 줄여 빌드 속도를 크게 향상시킬 수 있습니다.<br/>
4. 프로젝트의 .vcxproj 파일을 텍스트 편집기로 열어 아무 PropertyGroup 사이에 새로운 PropertyGroup 을 만들고 내부에 EnableUnitySupport 태그 삽입 및 true 설정을 추가하여 유니티 빌드를 활성화할 수 있습니다.<br/>
5. 여기서 '씬'은 게임의 각 화면이나 단계를 나타내는 개념으로 사용됩니다. '스타트 씬'은 게임 시작 화면일 수 있고, '툴 씬'은 개발자가 게임 오브젝트를 배치하거나 속성을 편집하는 등의 작업을 수행할 수 있는 편집 환경을 의미합니다.<br/>
6. '타일'은 2D 게임 배경이나 맵을 구성하는 기본적인 단위 이미지 조각을 의미합니다. 기존 오브젝트는 주로 자기 중심 좌표를 기준으로 렌더링하지만, 타일은 자신의 좌상단 좌표를 기준으로 설정된 크기만큼 렌더링하는 방식으로 처리됩니다.<br/>
7. 여러 개의 작은 타일 이미지를 하나의 큰 텍스처 아틀라스로 관리하면 게임 시작 시 한 번의 텍스처 로딩만으로 모든 타일 이미지를 사용할 수 있어 메모리 사용 효율성을 높이고, 렌더링 호출 횟수를 줄여 성능을 향상시킬 수 있습니다.<br/>
8. 타일 텍스처 내의 타일 이미지를 격자 형태로 간주하고 각 이미지에 고유한 인덱스를 부여합니다. 이 인덱스를 이용하여 텍스처의 가로 타일 개수와 해당 인덱스의 몫과 나머지를 계산하여 텍스처 내 해당 타일 이미지의 좌표를 찾아 렌더링에 활용합니다.<br/>
9. 모달 방식의 윈도우는 활성화되면 사용자의 모든 입력을 해당 윈도우로만 집중시켜 다른 윈도우와 상호작용할 수 없도록 만듭니다. 반면, 모달리스 방식의 윈도우는 활성화되어 있어도 다른 윈도우와 동시에 상호작용할 수 있습니다.<br/>
10. 윈도우 프로시저는 특정 윈도우에서 발생하는 다양한 메시지를 처리하는 콜백 함수입니다. 다이얼로그 박스에서도 자체적인 윈도우 프로시저를 통해 사용자의 상호작용에 반응하고 다이얼로그의 동작 방식을 정의합니다.<br/>

에세이 형식 질문
---

1. 유니티 빌드의 장점과 단점을 설명하고, 실제 프로젝트에서 유니티 빌드를 적용할 때 고려해야 할 사항들을 논하시오.<br/>
2. 게임 개발 툴의 중요성을 강조하고, Win32API를 이용하여 간단한 툴 기능을 개발하는 과정에서 얻을 수 있는 이점과 한계를 분석하시오.<br/>
3. 타일 기반 2D 게임 개발에 있어서 타일 맵 에디터의 필요성을 설명하고, 효율적인 타일 관리 및 렌더링 전략에 대해 논하시오.<br/>
4. 모달 및 모달리스 윈도우의 특징을 비교 분석하고, 게임 개발 툴의 사용자 인터페이스 디자인 관점에서 각 방식이 적합한 사용 사례를 제시하시오.<br/>
5. 이 강의 자료를 바탕으로 자신만의 간단한 타일 맵 에디터의 주요 기능과 구현 방안을 설계하고, 개발 과정에서 발생할 수 있는 기술적인 어려움과 해결 방안을 예상하여 논하시오.<br/>

용어 해설
---

1. ***빌드*** (Build): 작성된 소스 코드를 컴퓨터가 실행할 수 있는 실행 파일(exe 등) 형태로 변환하는 과정. 컴파일, 링크 등의 단계를 포함합니다.<br/>
2. ***컴파일*** (Compile): 프로그래밍 언어로 작성된 소스 코드를 컴퓨터가 이해할 수 있는 기계어 또는 중간 코드로 변환하는 과정.<br/>
3. ***링크*** (Link): 컴파일된 여러 개의 오브젝트 파일과 라이브러리들을 연결하여 하나의 실행 파일 또는 라이브러리 파일을 생성하는 과정.<br/>
4. ***전방 선언*** (Forward Declaration): 특정 클래스나 구조체의 이름만 미리 선언하여 컴파일러에게 그 존재를 알리는 것. 헤더 파일의 불필요한 포함을 줄여 컴파일 시간을 단축시키는 데 사용됩니다.<br/>
5. ***헤더 파일*** (Header File): C/C++에서 함수, 클래스, 구조체 등의 선언을 포함하는 파일. 소스 파일에서 해당 기능을 사용하기 위해 #include 지시어를 통해 포함됩니다.<br/>
6. ***유니티 빌드*** (Unity Build): 여러 개의 C++ 소스 파일을 하나의 임시 소스 파일로 합쳐서 컴파일하는 빌드 방식. 컴파일러의 중복 작업을 줄여 빌드 속도를 향상시킵니다.<br/>
7. ***씬*** (Scene): 게임의 화면 또는 단계를 의미하는 논리적인 단위. 각각의 씬은 고유한 오브젝트 구성과 게임 로직을 가질 수 있습니다.<br/>
8. ***타일*** (Tile): 2D 게임 배경이나 맵을 구성하는 작은 이미지 단위. 규칙적인 격자 형태로 배치되어 맵을 형성합니다.<br/>
9. ***텍스처 아틀라스*** (Texture Atlas): 여러 개의 작은 텍스처 이미지를 하나의 큰 텍스처 이미지 파일에 모아 놓은 것. 렌더링 효율성을 높이는 데 사용됩니다.<br/>
10. ***인덱스*** (Index): 배열이나 컬렉션 내의 특정 요소를 식별하기 위해 사용되는 정수 값. 텍스처 아틀라스 내에서 특정 타일의 위치를 지정하는 데 사용될 수 있습니다.<br/>
11. ***모달*** (Modal): 윈도우가 활성화되면 다른 윈도우와의 상호작용을 막고 해당 윈도우에만 사용자 입력을 집중시키는 방식.<br/>
12. ***모달리스*** (Modeless): 윈도우가 활성화되어 있어도 다른 윈도우와 동시에 상호작용할 수 있는 방식.<br/>
13. ***윈도우 프로시저*** (Window Procedure): 윈도우에서 발생하는 메시지(이벤트)를 처리하는 콜백 함수. 각 윈도우는 자신만의 윈도우 프로시저를 가질 수 있습니다.<br/>
14. ***다이얼로그 박스*** (Dialog Box): 사용자에게 정보를 표시하거나 입력을 받기 위해 임시로 나타나는 작은 윈도우. 주로 모달 방식으로 사용됩니다.<br/>
15. ***컨트롤*** (Control): 윈도우 내에서 사용자와 상호작용할 수 있는 시각적인 요소 (버튼, 텍스트 입력 상자 등).<br/>
